二叉搜索树的删除

发表于 2020-02-15 分类于 数据结构

二叉搜索树的删除：

在进行二叉搜索树元素的删除的时候要注意一个前提，即不影响二叉搜索树的性质。所以在实际删除过程中分为三种情况讨论。

1、如果要删除的元素是叶节点，则直接删除，再修改其父节点指针–置为NULL。

2、要删除的节点只有一个孩子节点：将其父节点的指针指向要删除节点的孩子节点。

3、要删除的节点有左右两棵子树：此时有有两种解法，一种是取右子树中最小的值替代此节点，第二种是取左子树中最大的值来替代。

伪代码：

BinTree FindMin(BinTree BST)//寻找最小值函数
{
    if(!BST)
    return NULL;
    else if(!BST->left)//找到最左边
    return BST;
    else return FindMin(BST->left);//沿着左分支继续查找
}

BinTree Delete(Element X,BinTree BST)//删除函数
{
    BinTree Tmp;
    if(!BST)
    {
        printf("未找到要删除的元素");
    }
    else if(X<BST->Data)
    BST->left = Delete(X,BST->left);//左子树递归删除
    else if(X>BST->Data)
    BST->right = Delete(X,BST->right);//右子树递归删除
    else//这条分支代表着找到了要删除的节点之后的操作
    {
        if(BST->right && BST->left)//被删除的节点有左右两个子节点
        {
            Tmp = FindMin(BST->right);
            BST->Data = Tmp->Data;//取右子树的最小值来代替
            BST->right = Delete(Tmp->Data,BST->right);//将右子树最小值删除
        }
        else //被删除节点有一个或者没有子节点
        {
            Tmp = BST;
            if(!BST->left)//如果没有左孩子节点或没有孩子节点
            {
                BST = BST->right;
            }
            if(!BST->right)//如果没有右孩子节点或没有孩子节点
            {
                BST = BST->left;
            }
            free(Tmp);
        }
    }
        return BST;
}

代码的个人理解：

首先要明白BinTree Delete(Element X,BinTree BST)函数的具体含义，他的含义是给他一个要删除的元素和根节点，函数实现删除此元素并且返回根节点。
所以在比较过程中，如果发现要删除的元素大于根节点，那么删除操作就可以在右子树中进行，具体体现在BST->right = Delete(X,BST->right)语句中。我将这种操作类比为接骨操作，右臂断了，那我就去右臂中找到断了的地方并修复（这边就是删除操作），修复之后再将骨头接上。
