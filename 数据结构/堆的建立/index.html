堆的建立

发表于 2020-02-26 分类于 数据结构

最大堆的建立：

在建堆的时候，往往有两种建堆方法：

方法一：

部分代码如下：

void InsertHeap(Pheap heap,int data)
{
int i;
if(IsFull(heap))
{
    printf("heap is full\n");
    return ;
}
else
{
    i = ++heap->size;
    for(;data>(heap->tree[i/2]);i=i/2)
    {
        heap->tree[i] = heap->tree[i/2];
    }
    heap->tree[i] = data;        
}
}

这种方法是通过一个一个的插入来建堆,但这时他的时间复杂度就为O(NlogN)，算法效率不高（建堆过程中一个要插入N个元素，每次插入的时间复杂度就是树的高度，即logN）。

方法二：

完整代码如下：

#include<stdio.h>
#include<stdlib.h>
typedef struct heap* Pheap;
struct heap{
int *tree;
int size;
int capacity;
};
Pheap HeapInit(int maxsize)
{
    Pheap myheap;
    myheap = (Pheap)malloc(sizeof(struct heap));
    myheap->tree = (int*)malloc((maxsize+1)*sizeof(int));
    myheap->capacity = maxsize;
    myheap->tree[0] = 10000;
    myheap->size = 0;
    return myheap;
}
void DownAdjust(int low,int high,Pheap heap)
{
    int child;
    int temp;
    int root = low;
    temp = heap->tree[root];
    while((root*2)<=high)//说明儿子存在 
    {
        child = root*2;//左儿子 

        if(root*2<high)//说明左右儿子都存在 
        {
            if(heap->tree[child+1]>heap->tree[child])
            {
                child = child+1;
            }
        }

        if(temp>heap->tree[child])//如果比左右儿子中最大的还要大，则说明找到存放的位置 
        {
            break;
        }

        else
        {
            heap->tree[root] = heap->tree[child];
            root = child;
            continue;
        }
    }
heap->tree[root] = temp;//跳出循环有两种情况，一种是无左右儿子，一种是找不到儿子比他还要大，这两者都是找到了恰当的位置 
    }
int main()
{
    int i;
Pheap heap = HeapInit(10);
heap->tree[1] = 85;
heap->tree[2] = 55;
heap->tree[3] = 82;
heap->tree[4] = 57;
heap->tree[5] = 68;
heap->tree[6] = 92;
heap->tree[7] = 99;
heap->tree[8] = 98;
heap->tree[9] = 66;
heap->tree[10] = 56;    
for(i=(heap->capacity)/2;i>=1;i--)
{
    DownAdjust(i,heap->capacity,heap);
}    
for(i=1;i<=heap->capacity;i++)
{
    printf("%d ",heap->tree[i]);
}    
return 0;    
}

这个方法的思路是：
（1）将N个元素按输入顺序存入，先满足完全二叉树的结构特性
（2）调整各结点位置，以满足最大堆的有序特性
这时他的时间复杂度为O(N)。

证明：

（1）具有n个元素的平衡二叉树，树的高度为logN，设为h。
（2）由于是由下往上建立堆（对于单个元素来讲其实可以理解为从上往下走，只是单个元素的选择是自右往左，自下往上），在调整上层元素的并不需要同下层所有的元素作比较，只要和其中之一的分支做比较，每一次比较都只要一次线性运算就可以确定大根，那么他比较的次数就是树的高度减去当前节点的高度，设他这一次的高度为x，那么需要比较的结点个数就是2^x个，所以第x层计算量就为2^x*(h-x)
(3)所以计算出时间复杂度为O(N)
